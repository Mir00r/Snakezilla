## 1. Firebase Project Setup

### Step 1: Create a Firebase Project

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Click **"Create a project"**
3. Enter project name: `snakezilla`
4. Enable **Google Analytics** (recommended for game analytics)
5. Select or create a Google Analytics account
6. Click **"Create project"**

### Step 2: Install Firebase CLI

```bash
# Install Firebase CLI globally
npm install -g firebase-tools

# Login to Firebase
firebase login

# Verify installation
firebase projects:list
```

```bash
# Install FlutterFire CLI
```

## 2. Flutter Firebase Integration

### Step 1: Configure Firebase for Your Flutter App

Run FlutterFire CLI in your project root:

```bash
flutterfire configure --project=your-firebase-project-id
```

This command will:
- Register your Android, iOS, and Web apps in the Firebase project
- Download `google-services.json` (Android) and `GoogleService-Info.plist` (iOS)
- Generate `lib/firebase_options.dart` with all platform configurations

### Step 2: Add Firebase Dependencies

Add these to `pubspec.yaml`:

```yaml
dependencies:
  # Core Firebase
  firebase_core: ^2.25.0

  # Analytics (game event tracking)
  firebase_analytics: ^10.8.0

  # Authentication (optional, for online features)
  firebase_auth: ^4.17.0

  # Firestore (online leaderboard, social features)
  cloud_firestore: ^4.15.0

  # Hosting is configured via CLI, no package needed

  # Optional: Push notifications for retention
  firebase_messaging: ^14.7.0

  # Optional: Remote Config for live-ops
  firebase_remote_config: ^4.3.0
```

Then install:

```bash
flutter pub get
```

### Step 3: Initialize Firebase in Your App

Update `lib/main.dart`:

```dart
import 'package:firebase_core/firebase_core.dart';
import 'firebase_options.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();

  // Initialize Firebase
  await Firebase.initializeApp(
    options: DefaultFirebaseOptions.currentPlatform,
  );

  runApp(const ProviderScope(child: SnakezillaApp()));
}
```

#### Android (`android/app/build.gradle`)

## 3. Firebase Analytics

Firebase Analytics lets you track game events, player behavior, and retention metrics.

### Setup

Create `lib/shared/services/firebase_analytics_service.dart`:

```dart
import 'package:firebase_analytics/firebase_analytics.dart';

class FirebaseAnalyticsService {
  static final FirebaseAnalytics _analytics = FirebaseAnalytics.instance;

  static FirebaseAnalyticsObserver get observer =>
      FirebaseAnalyticsObserver(analytics: _analytics);

  // ── Game Events ──────────────────────────────────────

  /// Log when a game session starts
  static Future<void> logGameStart({
    required String gameMode,
    required String difficulty,
    required String skin,
    String? pet,
    String? world,
  }) async {
    await _analytics.logEvent(
      name: 'game_start',
      parameters: {
        'game_mode': gameMode,
        'difficulty': difficulty,
        'skin': skin,
        'pet': pet ?? 'none',
        'world': world ?? 'default',
      },
    );
  }

  /// Log when a game session ends
  static Future<void> logGameEnd({
    required String gameMode,
    required int score,
    required int snakeLength,
    required int maxCombo,
    required int kills,
    required int durationSeconds,
  }) async {
    await _analytics.logEvent(
      name: 'game_end',
      parameters: {
        'game_mode': gameMode,
        'score': score,
        'snake_length': snakeLength,
        'max_combo': maxCombo,
        'kills': kills,
        'duration_seconds': durationSeconds,
      },
    );
  }

  // ── Economy Events ───────────────────────────────────

  /// Log coin spending
  static Future<void> logSpendCoins({
    required int amount,
    required String itemType, // 'skin', 'pet', 'world', 'gift'
    required String itemId,
  }) async {
    await _analytics.logSpendVirtualCurrency(
      itemName: itemId,
      virtualCurrencyName: 'coins',
      value: amount.toDouble(),
    );
  }

  /// Log coin earning
  static Future<void> logEarnCoins({
    required int amount,
    required String source, // 'game', 'daily_reward', 'spin_wheel', 'mission', 'gift'
  }) async {
    await _analytics.logEarnVirtualCurrency(
      virtualCurrencyName: 'coins',
      value: amount.toDouble(),
    );
  }

  // ── Progression Events ──────────────────────────────

  /// Log level up
  static Future<void> logLevelUp(int level) async {
    await _analytics.logLevelUp(level: level);
  }

  /// Log achievement unlock
  static Future<void> logAchievementUnlocked(String achievementId) async {
    await _analytics.logUnlockAchievement(id: achievementId);
  }

  /// Log prestige
  static Future<void> logPrestige(int prestigeLevel) async {
    await _analytics.logEvent(
      name: 'prestige',
      parameters: {'prestige_level': prestigeLevel},
    );
  }

  /// Log rank change
  static Future<void> logRankChange(String newRank, int rankPoints) async {
    await _analytics.logEvent(
      name: 'rank_change',
      parameters: {
        'new_rank': newRank,
        'rank_points': rankPoints,
      },
    );
  }

  // ── Engagement Events ────────────────────────────────

  /// Log daily reward claimed
  static Future<void> logDailyRewardClaimed(int day) async {
    await _analytics.logEvent(
      name: 'daily_reward_claimed',
      parameters: {'reward_day': day},
    );
  }

  /// Log spin wheel used
  static Future<void> logSpinWheel(String prize) async {
    await _analytics.logEvent(
      name: 'spin_wheel',
      parameters: {'prize': prize},
    );
  }

  /// Log tournament entry
  static Future<void> logTournamentEntry(String tournamentType) async {
    await _analytics.logEvent(
      name: 'tournament_entry',
      parameters: {'tournament_type': tournamentType},
    );
  }

  // ── Retention Events ─────────────────────────────────

  /// Log app open (automatic with Firebase, but custom data helps)
  static Future<void> logAppOpen() async {
    await _analytics.logAppOpen();
  }

  /// Log comeback (returning after absence)
  static Future<void> logComeback(int daysAway) async {
    await _analytics.logEvent(
      name: 'comeback',
      parameters: {'days_away': daysAway},
    );
  }

  // ── User Properties ──────────────────────────────────

  /// Set player level as user property for segmentation
  static Future<void> setPlayerLevel(int level) async {
    await _analytics.setUserProperty(name: 'player_level', value: '$level');
  }

  /// Set player rank for segmentation
  static Future<void> setPlayerRank(String rank) async {
    await _analytics.setUserProperty(name: 'player_rank', value: rank);
  }

  /// Set prestige level for segmentation
  static Future<void> setPrestigeLevel(int prestige) async {
    await _analytics.setUserProperty(name: 'prestige_level', value: '$prestige');
  }
}
```

### Key Metrics to Track in Firebase Console

| Metric | Event | Why It Matters |
|---|---|---|
| **DAU / MAU** | Automatic | Core engagement metric |
| **Session Duration** | `game_end.duration_seconds` | Are games too short/long? |
| **Games Per Session** | `game_start` count per session | How engaged are players? |
| **Mode Popularity** | `game_start.game_mode` | Which modes need balancing? |
| **Retention (D1/D7/D30)** | Automatic | Long-term health indicator |
| **Churn Points** | `game_end.score` distribution | Where do players give up? |
| **Economy Balance** | `earn_virtual_currency` vs `spend_virtual_currency` | Is the economy healthy? |
| **Feature Adoption** | Custom events | Are new features used? |

### Setting Up Funnels (Firebase Console)

1. Go to **Analytics > Events** in Firebase Console
2. Mark key events as **Conversion events**:
   - `game_start` (first game)
   - `level_up` (progression)
   - `prestige` (deep engagement)
   - `tournament_entry` (competitive engagement)
3. Create funnels under **Explore > Funnel Analysis**:
   - **New Player Funnel**: `first_open` → `game_start` → `game_end` (score > 0) → `level_up`
   - **Retention Funnel**: `app_open` → `game_start` → `daily_reward_claimed`
   - **Monetization Path** (future): `game_start` → `spend_virtual_currency`

## 4. Firebase Authentication

Authentication enables online features like cloud saves, global leaderboards, and social features.

### Anonymous Auth (Recommended for Games)

Anonymous auth lets players use online features without creating an account, then optionally link to Google/Apple later.

#### Enable in Firebase Console

1. Go to **Authentication > Sign-in method**
2. Enable **Anonymous** sign-in
3. (Optional) Enable **Google** and **Apple** sign-in for account linking

#### Implementation

Create `lib/shared/services/auth_service.dart`:

```dart
import 'package:firebase_auth/firebase_auth.dart';

class AuthService {
  static final FirebaseAuth _auth = FirebaseAuth.instance;

  /// Get current user (null if not signed in)
  static User? get currentUser => _auth.currentUser;

  /// Get user ID (creates anonymous account if needed)
  static Future<String> getUserId() async {
    if (_auth.currentUser == null) {
      await _auth.signInAnonymously();
    }
    return _auth.currentUser!.uid;
  }

  /// Check if user is anonymous
  static bool get isAnonymous => _auth.currentUser?.isAnonymous ?? true;

  /// Link anonymous account to Google
  static Future<UserCredential?> linkWithGoogle() async {
    try {
      final GoogleAuthProvider googleProvider = GoogleAuthProvider();
      return await _auth.currentUser?.linkWithProvider(googleProvider);
    } catch (e) {
      print('Failed to link Google account: $e');
      return null;
    }
  }

  /// Sign out
  static Future<void> signOut() async {
    await _auth.signOut();
  }

  /// Listen to auth state changes
  static Stream<User?> get authStateChanges => _auth.authStateChanges();
}
```

### Firestore Service Implementation

Create `lib/shared/services/firestore_service.dart`:

```dart
import 'package:cloud_firestore/cloud_firestore.dart';

class FirestoreService {
  static final FirebaseFirestore _db = FirebaseFirestore.instance;

  // ── Leaderboard ──────────────────────────────────────

  /// Submit a score to the global leaderboard
  static Future<void> submitScore({
    required String playerId,
    required String playerName,
    required int score,
    required String gameMode,
    required int snakeLength,
    required int maxCombo,
    required int kills,
    required String skin,
    required int level,
    required String rank,
  }) async {
    await _db.collection('leaderboard').add({
      'playerId': playerId,
      'playerName': playerName,
      'score': score,
      'gameMode': gameMode,
      'snakeLength': snakeLength,
      'maxCombo': maxCombo,
      'kills': kills,
      'skin': skin,
      'level': level,
      'rank': rank,
      'timestamp': FieldValue.serverTimestamp(),
      'platform': _getPlatform(),
    });
  }

  /// Get top scores for a game mode
  static Future<List<Map<String, dynamic>>> getTopScores({
    required String gameMode,
    int limit = 50,
  }) async {
    final snapshot = await _db
        .collection('leaderboard')
        .where('gameMode', isEqualTo: gameMode)
        .orderBy('score', descending: true)
        .limit(limit)
        .get();

    return snapshot.docs.map((doc) => doc.data()).toList();
  }

  /// Get a player's best score for a mode
  static Future<int> getPlayerBestScore({
    required String playerId,
    required String gameMode,
  }) async {
    final snapshot = await _db
        .collection('leaderboard')
        .where('playerId', isEqualTo: playerId)
        .where('gameMode', isEqualTo: gameMode)
        .orderBy('score', descending: true)
        .limit(1)
        .get();

    if (snapshot.docs.isEmpty) return 0;
    return snapshot.docs.first.data()['score'] as int;
  }

  // ── Player Profile ───────────────────────────────────

  /// Create or update player profile
  static Future<void> updatePlayerProfile({
    required String userId,
    required String name,
    required int level,
    required String rank,
    required int prestigeLevel,
    required int totalGamesPlayed,
    required int highScore,
  }) async {
    await _db.collection('players').doc(userId).set({
      'name': name,
      'level': level,
      'rank': rank,
      'prestigeLevel': prestigeLevel,
      'totalGamesPlayed': totalGamesPlayed,
      'highScore': highScore,
      'lastActive': FieldValue.serverTimestamp(),
    }, SetOptions(merge: true));
  }

  /// Get player profile
  static Future<Map<String, dynamic>?> getPlayerProfile(String userId) async {
    final doc = await _db.collection('players').doc(userId).get();
    return doc.data();
  }

  // ── Helpers ──────────────────────────────────────────

  static String _getPlatform() {
    // Simplified — in production use Platform.isAndroid etc.
    return 'unknown';
  }
}
```

### Creating Firestore Indexes

Some queries require composite indexes. Firebase will show an error link when you first run a query that needs one. Click the link or create them manually:

1. Go to **Firestore > Indexes** in Firebase Console
2. Add these composite indexes:

| Collection | Fields | Order |
|---|---|---|
| `leaderboard` | `gameMode` ASC, `score` DESC | Query scope: Collection |
| `leaderboard` | `playerId` ASC, `gameMode` ASC, `score` DESC | Query scope: Collection |
| `tournaments.entries` | `score` DESC | Query scope: Collection Group |

## 7. Firebase Cloud Messaging (Push Notifications)

Use FCM to send retention notifications (daily rewards reminder, tournament starting, comeback incentive).

### Enable in Firebase Console

1. Go to **Cloud Messaging** in Firebase Console
2. For iOS: Upload your APNs key (from Apple Developer account)
3. For Android: No additional setup needed

### Implementation

```dart
import 'package:firebase_messaging/firebase_messaging.dart';

class PushNotificationService {
  static final FirebaseMessaging _messaging = FirebaseMessaging.instance;

  static Future<void> initialize() async {
    // Request permission (iOS)
    final settings = await _messaging.requestPermission(
      alert: true,
      badge: true,
      sound: true,
    );

    if (settings.authorizationStatus == AuthorizationStatus.authorized) {
      // Get FCM token for this device
      final token = await _messaging.getToken();
      print('FCM Token: $token');
      // Save token to Firestore for targeted notifications
    }

    // Handle foreground messages
    FirebaseMessaging.onMessage.listen((RemoteMessage message) {
      print('Foreground message: ${message.notification?.title}');
      // Show in-app notification
    });

    // Handle background message tap
    FirebaseMessaging.onMessageOpenedApp.listen((RemoteMessage message) {
      print('Opened from notification: ${message.data}');
      // Navigate to relevant screen
    });
  }

  /// Subscribe to topic-based notifications
  static Future<void> subscribeToTopics() async {
    await _messaging.subscribeToTopic('daily_rewards');
    await _messaging.subscribeToTopic('tournaments');
    await _messaging.subscribeToTopic('events');
  }
}
```

## 8. Firebase Remote Config

Use Remote Config to adjust game parameters without releasing an update — perfect for balancing, seasonal events, and A/B testing.

### Setup in Firebase Console

1. Go to **Engage > Remote Config**
2. Add parameters:

| Parameter | Default Value | Description |
|---|---|---|
| `daily_coin_multiplier` | `1.0` | Multiply daily reward coins (set to 2.0 for events) |
| `max_tournament_attempts` | `3` | Daily tournament attempts |
| `welcome_back_bonus_coins` | `100` | Comeback reward base coins |
| `featured_skin` | `neon` | Showcase skin on home screen |
| `event_active` | `false` | Toggle seasonal events |
| `event_name` | `Summer Sizzle` | Current event name |
| `event_coin_bonus` | `1.5` | Event coin multiplier |
| `maintenance_mode` | `false` | Show maintenance screen |

### Implementation

```dart
import 'package:firebase_remote_config/firebase_remote_config.dart';

class RemoteConfigService {
  static final FirebaseRemoteConfig _config = FirebaseRemoteConfig.instance;

  static Future<void> initialize() async {
    await _config.setDefaults({
      'daily_coin_multiplier': 1.0,
      'max_tournament_attempts': 3,
      'welcome_back_bonus_coins': 100,
      'featured_skin': 'neon',
      'event_active': false,
      'event_name': 'Summer Sizzle',
      'event_coin_bonus': 1.5,
      'maintenance_mode': false,
    });

    // Fetch with a short cache for development, longer for production
    await _config.setConfigSettings(RemoteConfigSettings(
      fetchTimeout: const Duration(minutes: 1),
      minimumFetchInterval: const Duration(hours: 1), // 12 hours for production
    ));

    await _config.fetchAndActivate();
  }

  // ── Getters ──────────────────────────────────────────

  static double get dailyCoinMultiplier =>
      _config.getDouble('daily_coin_multiplier');

  static int get maxTournamentAttempts =>
      _config.getInt('max_tournament_attempts');

  static int get welcomeBackBonusCoins =>
      _config.getInt('welcome_back_bonus_coins');

  static String get featuredSkin =>
      _config.getString('featured_skin');

  static bool get isEventActive =>
      _config.getBool('event_active');

  static String get eventName =>
      _config.getString('event_name');

  static double get eventCoinBonus =>
      _config.getDouble('event_coin_bonus');

  static bool get isMaintenanceMode =>
      _config.getBool('maintenance_mode');
}
```

## 9. Security Rules

### Firestore Security Rules

Go to **Firestore > Rules** and set:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Leaderboard: anyone can read, authenticated users can write their own scores
    match /leaderboard/{scoreId} {
      allow read: if true;
      allow create: if request.auth != null
                    && request.resource.data.playerId == request.auth.uid
                    && request.resource.data.score is int
                    && request.resource.data.score >= 0
                    && request.resource.data.score <= 999999;
      allow update, delete: if false;  // Scores are immutable
    }

    // Player profiles: users can read all, write only their own
    match /players/{userId} {
      allow read: if true;
      allow write: if request.auth != null && request.auth.uid == userId;
    }

    // Tournament entries: authenticated players only
    match /tournaments/{tournamentId} {
      allow read: if true;

      match /entries/{userId} {
        allow read: if true;
        allow write: if request.auth != null && request.auth.uid == userId;
      }
    }

    // Gifts: sender can create, recipient can update (claim)
    match /gifts/{giftId} {
      allow read: if request.auth != null
                  && (resource.data.fromUserId == request.auth.uid
                      || resource.data.toUserId == request.auth.uid);
      allow create: if request.auth != null
                    && request.resource.data.fromUserId == request.auth.uid;
      allow update: if request.auth != null
                    && resource.data.toUserId == request.auth.uid
                    && request.resource.data.claimed == true;
      allow delete: if false;
    }
  }
}
```

### Key Security Principles

1. **Never trust the client** — validate all data in security rules
2. **Score validation** — cap scores at a reasonable maximum to prevent cheating
3. **User isolation** — players can only write their own data
4. **Immutable scores** — once submitted, leaderboard entries cannot be edited
5. **Rate limiting** — consider using Firebase App Check for additional protection

### Enable Firebase App Check (Anti-Abuse)

1. Go to **App Check** in Firebase Console
2. Register your apps:
   - **Android**: Use Play Integrity (recommended) or SafetyNet
   - **iOS**: Use App Attest (iOS 14+) or DeviceCheck
   - **Web**: Use reCAPTCHA v3
3. Enforce App Check on Firestore:
   - Go to **Firestore > App Check** and click **Enforce**

## 10. Troubleshooting

### Common Issues

| Issue | Solution |
|---|---|
| `No Firebase App '[DEFAULT]' has been created` | Ensure `Firebase.initializeApp()` is called in `main()` before `runApp()` |
| `google-services.json not found` | Run `flutterfire configure` again |
| `MissingPluginException` | Run `flutter clean && flutter pub get`, then rebuild |
| `Firestore permission denied` | Check security rules — ensure user is authenticated |
| `Index not found` error | Click the error's link to auto-create the Firestore index |
| iOS build fails with CocoaPods | Run `cd ios && pod install --repo-update && cd ..` |
| Web CORS errors | Ensure Firebase config is in `web/index.html` |
| Analytics events not showing | Events can take up to 24 hours to appear in Firebase Console. Use **DebugView** for real-time testing |

### Enable Analytics Debug Mode

For real-time event verification:

```bash
# Android
adb shell setprop debug.firebase.analytics.app com.yourcompany.snakezilla

# iOS (add to Xcode scheme arguments)
-FIRDebugEnabled

# Web (in browser console)
firebase.analytics().setAnalyticsCollectionEnabled(true);
```

Then check **Analytics > DebugView** in Firebase Console.

### Local Emulator Suite

For development without affecting your production Firebase project:

```bash

firebase emulators:start

# Emulator UI available at http://localhost:4000
# Firestore emulator: localhost:8080
# Auth emulator: localhost:9099
```

Update your app to use emulators in debug mode:

```dart
if (kDebugMode) {

  await FirebaseAuth.instance.useAuthEmulator('localhost', 9099);
}
```
